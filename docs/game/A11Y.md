# Pony Forge Game — Accessibility Spec

Target: WCAG 2.1 AA. Game must be fully playable without a mouse, with screen
reader support, reduced-motion fallbacks, and visible focus throughout.

## 1. Keyboard Play

The entire game loop is keyboard-reachable. No interaction depends on hover,
drag, or pointer precision.

| Key            | Action                                                  |
| -------------- | ------------------------------------------------------- |
| `P`            | Start game from idle. Resume from pause.                |
| `Esc`          | Pause game and enter chill mode (no timer, no scoring). |
| `Tab`          | Move focus between horses, target card, toggles.        |
| `Shift+Tab`    | Reverse focus order.                                    |
| `Enter`        | Activate focused horse (lock as current guess target).  |
| `Space`        | Submit current guess.                                   |
| `1`–`9`, `0`   | Apply hat 1–10 to focused horse (existing binding).     |
| `-`, `=`       | Apply hats 11 and 12 (existing binding).                |
| `M`            | Toggle mute.                                            |
| `N`            | Toggle night mode.                                      |
| `?`            | Open keyboard help overlay.                             |

Tab order: skip-link → mode toggle → mute toggle → help → target card →
horse 1..N → submit. The order matches visual reading order (top-to-bottom,
left-to-right). Hat number keys work without focus on a horse only when
exactly one horse is in the "active guess" slot; otherwise the focused horse
receives the hat.

## 2. Screen Reader Announcements

A single visually-hidden `aria-live` region (`#a11y-status`) hosts game
state updates. A second region (`#a11y-alerts`, `aria-live="assertive"`)
hosts urgent events only.

| Event                | Region    | Politeness  | Example text                        |
| -------------------- | --------- | ----------- | ----------------------------------- |
| Round start          | status    | polite      | "Round 3. Find the horse in the top hat." |
| Score change         | status    | polite      | "Score 240, plus 20."               |
| Combo milestone      | status    | polite      | "Combo three."                      |
| Heart lost           | alerts    | assertive   | "Wrong. Two hearts left."           |
| Game over            | alerts    | assertive   | "Game over. Final score 480."       |
| Pause / chill mode   | status    | polite      | "Paused. Chill mode."               |
| Mute toggled         | status    | polite      | "Sound off." / "Sound on."          |

Rules:
- Only post the delta on score change ("plus 20"), not the full history.
- Debounce live region updates to one message per 500ms; coalesce rapid
  scoring into a single "plus N" message.
- Never put decorative emoji or hat icons in the live region — screen
  readers will read filenames or alt text. Use the hat name ("top hat",
  "cowboy hat", "party hat") from the hat catalog.

## 3. prefers-reduced-motion

Honor `@media (prefers-reduced-motion: reduce)` everywhere animation is
purely decorative.

Skip when reduced motion is requested:
- `+score` floating number → render score delta in place for 600ms then fade
  via opacity step (no transform).
- Combo flash → swap to a static colored badge for the same duration.
- Game-over fade/zoom → instant transition to the modal.
- Horse idle bobbing, hat shimmer, paper texture parallax.

Keep (information-bearing motion):
- Timer bar drain. The bar still empties linearly so the time-remaining
  affordance is preserved. Disable the pulse-blink at the last 3 seconds —
  rely on color shift (oxblood → darker oxblood) and the live region
  warning "Three seconds left" instead.
- Heart removal: replace the shake with an instant state change plus
  assertive announcement.

Implementation: a single `data-reduced-motion="true"` attribute on `<html>`
toggled by the media query, with CSS rules gated on
`html[data-reduced-motion="true"] .anim-...`.

## 4. Color Contrast

Palette tokens (existing): `--paper #f4ecd8`, `--oxblood #6b1f1f`,
`--ink #1a1a1a`, `--night-bg #161310`, `--night-paper #2a2520`.

| Pair                              | Ratio   | Status     |
| --------------------------------- | ------- | ---------- |
| `--ink` on `--paper` (body)       | 13.9:1  | AAA        |
| `--oxblood` on `--paper` (body)   | 7.8:1   | AAA        |
| `--oxblood` on `--paper` (UI)     | 7.8:1   | AAA        |
| `--paper` on `--night-bg` (body)  | 11.2:1  | AAA        |
| Timer bar fill on night paper     | 4.6:1   | AA — verify |

Action items:
- Verify timer bar `--oxblood` against `--night-paper` at the half-empty
  state where it overlaps the track. Track must use `--ink` at 0.4 alpha,
  not pure transparency, to maintain 3:1 non-text contrast.
- Hat-ring overlay tint must keep horse silhouette ≥3:1 against background.
- Never encode state with color alone. Heart loss = icon change + count +
  announcement. Combo = badge text "x3" + color.

## 5. Focus Rings

Visible focus is mandatory. No `outline: none` without replacement.

```css
:focus-visible {
  outline: 2px solid var(--oxblood);
  outline-offset: 3px;
  border-radius: 4px;
}
html[data-night="true"] :focus-visible {
  outline-color: var(--paper);
}
```

Applied to: every horse card, target card, mode toggle, mute toggle, help
button, submit button, game-over modal buttons, restart link. The ring sits
outside the element so it does not occlude the hat ring on horses.

## 6. Toggles

Mute and mode toggles are real `<button>` elements.

```html
<button type="button"
        aria-pressed="false"
        aria-label="Mute sound effects">
  <span aria-hidden="true">[icon]</span>
</button>

<button type="button"
        aria-pressed="false"
        aria-label="Switch to night mode">
  <span aria-hidden="true">[icon]</span>
</button>
```

`aria-pressed` flips on activation. The accessible name updates to reflect
the next action ("Unmute", "Switch to day mode") when pressed. Both are
≥44×44 CSS px and reachable via Tab in the order defined above.

## 7. Game Over Modal

The modal is a focus trap.

- Element: `<div role="dialog" aria-modal="true" aria-labelledby="go-title">`.
- On open: save `document.activeElement` to `lastFocus`, move focus to the
  modal heading (`tabindex="-1"`), set `inert` on background.
- Tab cycles within the modal (Restart → Share → Close). Shift+Tab wraps.
- Esc closes the modal.
- On close: remove `inert`, restore focus to `lastFocus` (typically the
  start button or last-focused horse).
- Background page is not just visually hidden — it is `inert` so screen
  readers do not reach it.

## 8. Mobile and Touch

- All interactive controls are ≥44×44 CSS px (WCAG 2.5.5 AAA target). Hat
  selector buttons use a 44px tap area even when the visible glyph is
  smaller; padding fills the gap.
- The hat ring (radial selector around a horse) responds to `pointerdown`
  and supports both tap-to-toggle and drag-to-select. A long-press on a
  horse opens the ring; a second tap on a hat applies it; a tap outside
  closes the ring.
- Pinch-zoom is not blocked. The viewport meta tag must omit
  `user-scalable=no` and `maximum-scale`.
- Touch targets do not overlap. Minimum 8px gap between adjacent horses on
  the smallest supported width (320px).
- Orientation: game works in portrait and landscape. No orientation lock.

## 9. Semantic Structure

- Single `<h1>` per page ("Pony Forge").
- Game region: `<main aria-label="Pony Forge game">`.
- Score, hearts, timer wrapped in `<div role="status" aria-live="off">`
  with the live updates routed through the dedicated regions in §2 — the
  visible HUD itself is not a live region to avoid double-announcements.
- Horse cards: `<button>` (not `<div>`) with
  `aria-label="Horse 3, wearing top hat"` updated on hat change.
- Skip link: `<a href="#game" class="skip-link">Skip to game</a>` first in
  source order.
